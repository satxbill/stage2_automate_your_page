1.1 The basics of Web and HTML

Web Structure

The basic high level elements of "The Web" are user's computers running browsers, the Internet, and Servers.

HTML

HTML stands for HyperText Markup Language. HTML documents make up the majority of the content on the web. HTML documents contain text content which describes "what you see" and markup which describes "how it looks".

Tags and Elements

HTML documents are comprised of HTML Elements. HTML Elements are defined using HTML Tags. Tags typically have begin and end components, with void type tags being an exception only having a single component. An example of a void tag is the "br" tag.

Some tags have content, some have content and attributes. Some tags are inline, menaing they work 'inline' with other HTML elements. Some tags are block - which for now means they create an invisible block in the HTML document that can have attributes which we'll learn about later.

Divs, Spans, Classes & Ids...

Divs are mainly used to group items together to: 1) Logically organize our code, and 2) Group our items together so we can add styling to the whole block of code all at once. This avoids repetition because we don't have to single handedly assign the same styles to all of our elements inside our div block.

Since spans are inline-level elements, they are usually not used to group items together and are used to style parts of text or a few elements together without affecting the layout of the page.

Ids are used to uniquely identify an element in the entire web page. That means div id="welcome" should be the only element with the unique identifier: "welcome". There should not be any other element with the name "welcome".

We should use id for special cases to give us the ability to uniquely identify elements and should be used sparingly because having a lot of id's in a website will slow our browser down.

Classes are used to style a group or groups of elements similarly. Using classes enables the programmer to avoid repetition by grouping like elements and using a single CSS entry to define the style for those elements.

A Note about Repetition

The previous paragraph mentioned the use of classes as a way to avoid repetition in code (and a way to help ensure consistency in how the web page will look). Don't Repeat Yourself!! or "DRY"... Perhaps one of the most important principles of good coding. When you repeat yourself not only do you create more work for you, you increase the chances of introducing errors in your code and you make it that much harder for others to maintain, modify and debug your code.

Summary

HTML is simply made up of a series of rectangles we put on a page. We use certain elements to layout our rectangles in either block-level or inline-level to control the flow of our pages.

Divs are block-level and spans are inline-level and are used differently. Divs are used more for grouping items together and creating a layout structure and spans are used to style text and individual elements.

Ids should be used sparingly and are used to uniquely identify an element. Class is recommended over id to style similar elements together.

Using Divs, Spans, IDs and Classes helps provide strucuture to HTML code. Good code structure facilitates readability, understanding and maintainability.

2.1 Intro to Serious Programming

Programming

A computer is a machine that can execute a program. With the right program, a computer can do any mechanical computation you can imagine.

A program describes a very precise sequence of steps. Since the computer is just a machine, the program must give the steps in a way that can be executed mechanically. That is, the program can be followed without any thought.

A programming language is a language designed for producing computer programs. A good programming language makes it easy for humans to read and write programs that can be executed by a computer.

Python is a programming language. The programs that we write in the Python language will be the input to the Python interpreter, which is a program that runs on the computer. The Python interpreter reads our programs and executes them by following the rules of the Python language.

Why do we need programming languages?

Why do we need to invent and learn new languages, like Python, to program computers, rather than using natural languages like English or Mandarin?

  • Ambiguity - there are a lot of ways to interpret natural language.
  • Verbosity - natural languages are verbose.

There are many reasons why a designed language like Python is better for writing programs than a natural language like English. One problem with natural languages is that they are ambiguous. Hence, not everyone will interpret the same phrase the same way. To program computers, it is important that we know exactly what our programs mean, and that the computer will run them with the meaning we intended. Another problem with natural language is that they are very verbose. To say something with the level of precision needed for a computer to be able to follow it mechanically would require an awful lot of writing. We want our programs to be short so it is less work to write them, and so that it is easier to read and understand them.

Grammer

Compared to a natural language, like English, programming languages like Python adhere to a strict grammatical structure. In English, even if a phrase is written or spoken incorrectly, it can still be understood with the help of context or other cues. On the other hand, in a programming language like Python, the code must match the language grammar exactly. The Python interpreter has no idea what to do with input that is not in the Python language, so it produces an error.

Backus-Naur Form

The notation we used to describe the grammar is known as Bakus-Naur Form. It was introduced in the 1950s by John Backus, the lead designer of the Fortran programming language at IBM.

The purpose of Bakus-Naur Form is to describe a programming language in a simple and concise manner.

Grace Hopper (1906-1992)

Grace Hopper was a pioneer in computing, a Navy rear admiral and the creator of COBOL - one of the first programming languages. She built the first compiler (a program that takes as input a human readable programming language and outputs a program that computers can execute). Compilers take convert the human readable code into machine executable code all at once and typically on the the executable output from the compiler is loaded into memory when the program is actually run on a computer. Python is an interpreted language meaning the Python program converts the code one step at a time as the program runs.

2.2 Variables and Strings

Variables

A variable is a name that refers to a value. In Python, we can use any sequence of letters and numbers and underscores (_) when we want to make a variable name, so long as it does not start with a number. There is a Python convention that variable names should not start with a capital letter (for ease of readability). This is not a Python grammer rule, merely a best practice.

To introduce a new variable in Python we use an assigment statement, in the case of Python this is the "=" sign.

class="centered_text"> name = expression

After executing an assignment expression the name on the left of the "=" sign refers to the value on the right of the "=" sign. Once a value has been assigned to a variable name the name can be referred to anywhere in the code and it will mean the same thing as the value it refers to. Here is an expression using a variable to print out weight_in_grams * 10.

weight_in_grams = 100
print weight_in_grams * 10

Variables can be useful to programmers in many ways, here are a few of them:

  • Improving code readability by using names that make sense to people
  • Giving programmers ways to store the values of important dataL
  • Giving programmers a way to change the value of something (i.e count = count - 1)

String Basics

A string in Python is a sequence of characters surrounded by quotes, either single or double. The only requirement is that the string must start and end with the same type of quote. Double quotes are preferred as it allows you to include quotes as characters in the string.

You can use the "+" operator to concatenate strings:

<string> + <string>   outputs the concatenation of the two strings

You cannot use the "+" operator to combine strings and integers.

You an use the "*" operator to multiply strings. This essentially creates multiple copies of the same string and concatenates them.

print '!' * 12
!!!!!!!!!!!!

The above program multiplies the string by the integer to return 12 exclamation points.

Indexing Strings

When you want to select sub-sequences from strings it is called indexing. Use the square brackets [] to specify which part of the string you want.

< string > [ < expression > ]

For example, if we have the string 'udacity' and we want to select the character in the zero position (that is, the first character), we would write:

'udacity'[0]

For more details on indexing strings in Python please watch this video.

KEY POINT: The first character of a string in Python is at position 0 - not 1!!!

Selecting Sub-Sequences from Strings

You can select a sub-sequence of a string by designating a starting and ending position in the string.

< string > [ < expression > ]   returns a one character string

<string >[ <start expression >: <stop expression >]   returns a string that is a sub-sequence of the characters in the string, from the start position up to the character before the stop position. If the start position is missing the sub-sequence starts from the beginning of the string; if the stop position is missing the sub-sequence goes to the end of the string.

For more details on selecting sub-sequences of strings in Python please watch this video.

Finding Strings in Strings

The find method is a built in operation, or method, provided by Python, that operates on strings. The output of find is the position of the string where the specified sub-string is found

<search string >.find( <target string >)

If the target string is not found anywhere in the search string, then the output will be -1.

For more details on finding strings in strings using Python please watch this video.

Finding Strings with Numbers

In addition to passing in a target string to find, we can also pass in a number:

<search string >.find( <target string >, <number >)

The number input is the position in the search string where find will start looking for the target string. So, the output is a number giving the position of the first occurrence of the target string in the search string at or after the number input position. If there is no occurrence at or after that position, the output is -1.

2.3 Input ->Function ->Output

Functions

A function is a block of organized, reusable code that is used to perform a single, related action. Functions provide better modularity for your application and a high degree of code reusing. Functions may take inputs, do something with the inputs and then produce output.

For example a function called cube might take a number as input and return the cube of that number as output.

Making vs. Using a Function

Use the keyword def followed by the function name and and function parameters in parentheses to define a function. The code block within a function starts with a ":" and is indented. Use the return keyword to return a result from the function.

The following code could be the definition of a function called cube:

def cube(x):
answer = x * x * x
return answer

To use a function type the function name and any parameters in parenthesis:

print cube(3)
>>>27

Functions help avoid Repetition

Defining a function lets the programmer create sections of code one time that can be reused multiple times in a program.

The "return" Statement

The return statement in a Python function indicates what value the function will output. Without the return statement a Python function will return the value "None".

2.4 Control Flow & Loops: If and While

Equality Comparisons

The "==" operator is used for equality comparisons in Python - because the "=" operator is the assignment operator. a == b would be used to determine if a and b are equal whereas a = b would be used to assign the current value represented by the variable "b" to the variable "a".

"a" and "b" can also be Python expressions when used with the equality comparison operator.

The equality comparison operator == returns True if the expressions being compared are in fact, equal. Otherwise False is returned.

Comparison Operators

Python provides several operators for making comparisons:

  • < less than
  • > greater than
  • <= less than or equal to
  • >= greater than or equal to
  • == equal to
  • != not equal to

If Statements

If statements enable control of what code executes based on the result of a test expression.

if <TestExpression>:
<block>

In the above statement the code in the <block> runs only if the value of the test expression is True. The end of an if statement is determined by indentation similar to a procedure.

You can add an else clause to an if statement to provide alternate code to execute when the test expression is False.

if <TestExpression>:
<block>
else:
<block>

Or Expressions

The or expression give the logical or of two operands (expressions). If either expression evaluates to True the value of the or is True. If both expressions are Fales the value of the or is False.

<Expression> or <Expression>

Here are a few examples:

print True or False
True
print False or True
True
print True or True
True
print False or False
False

Composition

The idea of composition was introduced and defined as using procedures within procedures to get to a result. An example was to use two calls to the 'bigger' procedure to create the 'biggest' procedure

def bigger(a,b):
if a > b:
return a
else:
return b


def biggest(a,b,c):
return(bigger(bigger(a,b),c))

While Loops

Loops are a way to execute code repeatedly and are typically bounded by one or more test expressions. The syntax for the While loop is very similar to the if statement:

while <TestExpression>:
<block>

A while loop executes the block any number of times - as long as the test expression is True. Recall the if statement executes the block only once. As with the if statement the block to execute is determined by indentation.

If the test expression for a while loop is true the block executes. Then, the test expression is evaluated again - if it is still true the block executes again. This continues as long as the test expression evaluates to True. There is no guarantee that the test expression will ever evaluate to False. The loop could keep executing forever, which is known as an infinite loop.

Here is an example of a while loop:

i = 0
while i < 10:
print i
i = i + 1

The above code will execute as long as the test expression i < 10 is True. In this example the code will prints i and then adds 1 to i. The loop will repeat 10 times printing the numbers 0 through 9.

2.5 Debugging

Some thoughts on Debugging Strategies

  1. Use Google to search on the error messages you receive.
  2. When using sample code as a model for your own code make sure the sample code does what it says it does.
  3. Use care when using sample code to ensure you don't inherit bugs from it.
  4. Use print statements to print variables (intermediate results) at various locations in your code to make sure each part is working as designed.
  5. Keep and compare old versions of your code. Save working versions so you have something to go back to if you introduce too many bugs. Use somthing like Git to automatically version your code.

Process of Debugging

  1. Observe what is happening in your program by checking the state of the program in various places.
  2. form a hypothesis as to what your program is actually doing compared with what you want it to do.
  3. Make and run tests to confirm or reject your hypothesis; fix it if you now know what is causing the problem or observe more of your program states if you do not.

2.6 Structured Data: Lists &For Loops

Lists

Strings are considered a type of structured data in that you can access individual characters and operate on sub-sequences. Lists are similar to strings in that a list is a grouping of ‘things’ however, the ‘things’ in a list may be any Python value not just characters as with a string. Structured data such as a Python list has some advantages over strings in that data can easily be stored, queried, analyzed and updated within the list structure.

Mutability

Items in a list are also mutable - that is they can be changed. Strings and substrings are immutable - that is they cannot be changed. Once a string value is assigned to a variable any future assignment statements using that variable name will only change where the variable points - the original string values are not changed. Additionally you cannot use an assignment variable to update a substring of a string.

Lists vs. Strings

s = 'abcd'
l = ['a','b','c','d']

In the code above the first line initializes a variable 's' which refers to the string 'abcd'. the second line creates a list 'l' containing 4 elements.

t = s[0]
m = l[0]

The first statement above assigns the value 'a' to the variable 't'. 't' points to a location in memory containing 'a' - not the same string as 's'. The second statement assigns the value a to the variable 'l' - similar to the first statement.

s[0] = 'x'
l[0] = 'q'

The first statement above results in an error - you cannot assign a value to a substring in Python, that is because strings are immutable. The second statement updates the original list and changes the first item in the list to 'q' so now the list 'l' is ['q','b','c','d']. This demonstrates the Mutability of lists, i.e. they can be changed.

Nested Lists

Elements in a list can be of differing types - you can mix and match strings, numbers and even other lists! For example:

months = [["Jan", 31],["Feb", 28],["Mar", 31]]

Creates a list 'months' that contains three elements, each of which is a 2 element list.

Aliasing

Consider the following code:

p = ['H','e','l','l','o']
p[0] = 'Y'
q = p

After the last assignment q = p p and q refer to the same list: ['Y','e','l','l','o']. So, anything done to mutate that list changes the value that both variables point to. When multiple variables refer to the same object it is called aliasing.

List Operations

In this lesson we learned some of the built in operations on lists in Python: append, '+', and len.

Append. Append adds a new element to the end of a list. The append method mutates the list that it is invoked upon (it does not create a new list). Syntax:

<list>.append(<element;>)

Concatenation. The + operator when used with lists concatenates the lists creating a new list. It does not mutate the original lists.

p = [0, 1]
q = [2, 3]
pq = p + q

pq now refers to a new list: [0 ,1 ,2 ,3].

Length. The len operator is used to find out the length of an object. len works with any object that is a collection of things (including strings). The output of len is the number of elements in its input. Syntax:

len(<list>) -> <number>

Loops on Lists

We can use whileloops to loop through lists by setting an index counter variable and incrementing it to step through the list. However, Python provides an easier way to do this: the for loop. The syntax looks like this:

for <name> in <list>:
<block>

The loop goes through each element in the list, assigning it to the <name> and executing the <block>. For example, to print all elements in a list you could define the following procedure:

def print_all_elements(p):
for e in p:
print e

A few more operations you can do with lists include:

<list>.index(<value>) returns the first instance of <value> in the list. Returns and error if <value> is not in the list.

<value> in <list>) returns True if <value> is in the list, False if it is not.

<value> not in <list>) returns True if <value> is not in the list, False if it is.

2.7 How to Solve Problems

Rules for Solving Problems

  1. Don't Panic!!
  2. What are the inputs?
  3. What are the output?
  4. Work through some examples by hand.
  5. Try to develop a simple mechanical solution.
  6. Develop code incrementally and test as you go.

It's just that easy!

A bit more about the Rules...

Don't Panic. Seriously - relax. Don't rush into writing code, or even designing algorithms. Consider the problem statement holistically and think about how you can break the problem down into more manageable pieces.

Consider the Inputs. How are the inputs represented? What are their data types? Where you given any specifications to follow? etc...

Consider the Outputs. How should you produce the outputs? Print them? Use the return statement to return them? What data types should you use?

Work examples by hand. Look at sample expected inputs and work through by hand what the expected outputs would be for given inputs. No coding yet at this point.

Develop a Mechanical Solution Try working through a few cases as a human would solve the problem, for example. In the daysBetweenDates problem from the class - it is reasonable to think a human might resort to using a physical calendar to work through the problem by hand. Use whatever physical tools you might use based on the specific problem to work through various scenarios of inputs and expected outputs.

You can progress through this process by starting to create algorighms represented by pseudocode or rough Python code. Consider the mechanical steps you worked out previously and work on algorithms to solve each step of the mechanical process. Consider whether a simpler approach might be better for a computer than perhaps the approach a human would take.

Develop code Incrementally When you have a workable algorithm and have thought about how to adapt it for a computer. Write code in small bits that are easily testable separately. Building incrementally towards solving the problem.
